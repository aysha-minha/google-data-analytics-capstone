Cleaning steps (high level):
1. Open raw CSV in Excel.
2. Remove obviously invalid rows:
   - rides with blank start or end times
   - duplicate ride_id rows
3. Calculate ride_length = ended_at - started_at
4. Filter out rides with ride_length < 1 minute or > 24 hours
5. Add day_of_week = WEEKDAY(started_at, 2)
6. Save cleaned master file: August_data_cleaned.xlsx
7. Make a trimmed copy for Sheets: August_data_cleaned_trimmed.xlsx (keep only analysis columns)

Analysis steps (high level):
1. Upload trimmed file to Google Drive → Open in Google Sheets
2. Create pivots:
   - Number of rides by user type (COUNT of ride_id by member_casual)
   - Rides by day_of_week (COUNT of ride_id split by member_casual)
   - Bike type preference (COUNT of ride_id by rideable_type vs member_casual)
3. Export charts as PNG into visuals/

Outputs (saved):
- visuals/Number of Rides by User Type.png
- visuals/Number of Rides by Day of Week(Member vs Casual.png
- visuals/Bike Type Preference(Member vs Casual).png
- docs/Capstone_Report.docx (draft)
- final/Capstone_Report.pdf (final export)

Limitations & notes:
- Analysis uses August only (representative sample); full-year analysis requires SQL/R for scalability
- Excel formulas documented in scripts/excel_formulas.txt
